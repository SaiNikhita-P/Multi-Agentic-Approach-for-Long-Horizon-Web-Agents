# WMA + A² — Web Agents with World Models and Hierarchical Abstraction

This repository integrates [Web Agents with World Models (WMA)](https://github.com/kyle8581/WMA-Agents) with the [A² (Assimilation and Accommodation)](https://github.com/Xinyu-Pang/A2) framework for long-horizon web task reasoning on the [Mind2Web](https://arxiv.org/abs/2306.06070) benchmark.

---

## Overview

This approach integrates world-model-based planning with task-adaptive hierarchical abstraction to improve long-horizon web navigation. The WMA framework uses a fine-tuned world model to predict future webpage states and a value model to score candidate actions before execution. The A² framework augments the policy-generation stage with reusable semantic and episodic abstractions induced from previously solved trajectories — via **Assimilation** (extracting reusable procedural workflows from past experience) and **Accommodation** (adapting those abstractions to new tasks and environments). Together, they enable more grounded, context-aware action generation alongside future-state-driven planning.

The original WMA codebase targets the WebArena benchmark. Significant changes have been made in the `mind2web/` folder to adapt the pipeline for evaluation on Mind2Web.

## Environment Setup

Create a `.env` file inside `WMA_integrated_with_A2/WMA-Agents/` with the following:

```env
OPENAI_API_KEY=your_openai_api_key_here
DATASET=mind2web
```

> `DATASET=mind2web` is required to prevent the pipeline from defaulting to WebArena-specific logic.

---

## Dataset

Download the Mind2Web dataset into:

```
WMA_integrated_with_A2/WMA-Agents/mind2web/data/
```

Download both the **train** and **test** splits. The train split is used during offline abstraction induction; the test split is used for evaluation.

---

## Pipeline

### Step 1 — Build World Model Training Data

Extract raw state transitions from exemplar trajectories:

```bash
python WMA_integrated_with_A2/WMA-Agents/mind2web/scripts/build_wm_transitions.py
```

Reads from `data/memory/exemplars.json` and outputs `(intent, o_t, a_t, o_{t+1})` tuples to `data/transitions/wm_raw_transitions.jsonl`.

Then generate abstracted transitions:

```bash
python WMA_integrated_with_A2/WMA-Agents/mind2web/scripts/build_wm_abstractions.py
```

Produces transition-focused observation abstractions (UPDATED / ADDED / DELETED elements) to reduce DOM noise before training.

---

### Step 2 — Train the World Model

Fine-tunes `meta-llama/Meta-Llama-3-8B-Instruct` with LoRA adapters to learn environment dynamics `(intent, o_t, a_t) → ô_{t+1}`:

```bash
python WMA_integrated_with_A2/WMA-Agents/mind2web/scripts/train_world_model.py
```

Model is saved to `mind2web/models/world_model/`.

---

### Step 3 — Build the Value Dataset

Generate candidate actions and assign trajectory-progress-based rewards:

```bash
python WMA_integrated_with_A2/WMA-Agents/mind2web/scripts/generate_value_dataset_v2_1.py
```

Then augment each sample with world-model-predicted next observations:

```bash
python WMA_integrated_with_A2/WMA-Agents/mind2web/scripts/generate_value_dataset_v2_1_with_obs.py
```

Clean and deduplicate the dataset:

```bash
python WMA_integrated_with_A2/WMA-Agents/mind2web/scripts/clean_dataset.py
```

---

### Step 4 — Train the Value Model

Trains a frozen `meta-llama/Meta-Llama-3.1-8B-Instruct` backbone with a lightweight regression head on `(intent, o_t, a_t, ô_{t+1}) → reward`:

```bash
python WMA_integrated_with_A2/WMA-Agents/mind2web/scripts/train_value_model_v3.py
```

Model is saved to `mind2web/models/value_model_v3/`.

---

### Step 5 — Offline Abstraction Induction (A²)

Generate website-level and domain-level workflow abstractions from the Mind2Web train split. These serve as action-candidate context during inference:

```bash
python WMA_integrated_with_A2/WMA-Agents/mind2web/offline_induction.py
```

An `abstractions/` folder is created under `WMA_integrated_with_A2/WMA-Agents/` containing the induced website-level and domain-level workflow memories.

---

### Step 6 — Run the Evaluation Pipeline

From `WMA_integrated_with_A2/WMA-Agents/`:

```bash
python -m mind2web.run_mind2web \
  --data_dir mind2web/data \
  --branching_factor 3 \
  --vf_budget 5
```

| Argument | Description |
|---|---|
| `--vf_budget` | Number of candidate actions generated by the policy model |
| `--branching_factor` | Number of top candidates passed to the world model and value model |

> **Important:** Record the exact start time before running. Results are saved with timestamped filenames such as `0_20260317_183245.json`.

---

### Step 7 — Compute Scores

From `WMA_integrated_with_A2/WMA-Agents/mind2web/results/`:

```bash
python calc_score.py \
  --results_dir gpt-3.5-turbo \
  --start_time %Y%m%d_%H%M%S
```

Replace `%Y%m%d_%H%M%S` with the actual timestamp recorded at pipeline start (e.g., `20260317_183245`).

## Repository Structure

```
WMA_integrated_with_A2/
└── WMA-Agents/
    ├── .env                          # API key and dataset config (create manually)
    ├── abstractions/                 # Generated A² workflow memories (created at runtime)
    └── mind2web/

        ├── models/

        ├── results/                  # Timestamped evaluation outputs
```
